# 测试方案与报告

## 1. 测试计划
- 测试目标：验证已实现的核心功能——收支记录的增、删、改、查，以及统计接口返回正确数据。
- 测试策略：结合手工测试与简单的API调用验证。优先验证正常流程，再覆盖异常情况。

## 2. 测试用例（至少3个）

| 用例编号 | 功能 | 输入 | 预期输出 | 实际输出 |
| --- | --- | --- | --- | --- |
| TC1 | 新增记录（正常） | {type:'expense', amount:50, category:'餐饮', date:'2025-01-06', note:'午餐'} | 返回201，包含新记录id和字段 | （手动/自动执行后填写） |
| TC2 | 查询记录（时间范围） | GET /api/records?start=2025-01-01&end=2025-01-31 | 返回符合条件的记录数组 | （手动/自动执行后填写） |
| TC3 | 删除不存在的记录（异常） | DELETE /api/record/999999 | 返回404和错误信息 | （手动/自动执行后填写） |

## 3. 测试执行
- 执行环境：Windows + Python + 虚拟环境
- 操作步骤摘要：
  1. `python db_init.py` 初始化数据库。
  2. `python app.py` 启动服务。
  3. 使用浏览器或 Postman/脚本访问 API。

手动/自动测试结果：请在本地运行并记录“实际输出”列内容。可使用 `pytest` 或简单的 requests 脚本实现自动化（本示例未提供自动测试脚本，但鼓励扩展）。
